## Issues Identified & Resolved

### 1. Visual Regression (Disappearing Widgets)
*   **Symptom:** Tasks running logically (events logging) but widgets invisible after T1 block.
*   **Root Cause:** `AbstractPlugin` logic did not clear old widget references properly upon `start()`/restart, leading to "zombie" widgets or overlay issues.
*   **Fix:** Patched `src/python/vendor/openmatb/plugins/abstractplugin.py` to explicitly hide and clear `self.widgets` before `create_widgets()` is called in `start()`.

### 2. Communications Auto-solving in Unattended Mode
*   **Symptom:** Audio prompts heard, but frequencies changed instantly without user input (Auto-solver behavior).
*   **Root Cause:** The `run_openmatb.py` harness forced `automaticsolver=True` for *all* plugins in validation mode.
*   **Fix:** Modified `src/python/run_openmatb.py` to exclude the `communications` plugin from global auto-solver instantiation, preserving manual verification logic.
*   **Verification:** Added Requirement 7 to pipeline (checking for rapid frequency changes).

### 3. Resman State Persistence (Non-Reset)
*   **Symptom:** Fuel tank levels from T1 carried over to T2, causing immediate failures or inconsistent starting states.
*   **Root Cause:** Plugin instance parameters were mutated during runtime and not reset on `start()`.
*   **Fix:** Updated `src/python/vendor/openmatb/plugins/resman.py` to deep-copy initial parameters on `__init__` and restore them on `start()`.
*   **Verification:** Added Requirement 6 to pipeline (checking `tank-a-level` param at block start).

### 4. Schedule Bar Accumulation
*   **Symptom:** Schedule plugin bars behaved erratically on restarts.
*   **Fix:** Cleared `planning` dictionary in `src/python/vendor/openmatb/plugins/scheduling.py` on scenario load.
