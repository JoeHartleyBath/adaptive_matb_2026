# Lab notes — 2026-01-22

## Goal
End-to-end pilot dry run using OpenMATB with deterministic workload blocks, LSL markers, and external output logging; iterate on UX and input issues encountered during manual walk-through.

## Environment
- OS: Windows
- Runner: [src/python/run_openmatb.py](src/python/run_openmatb.py)
- OpenMATB vendored submodule: [src/python/vendor/openmatb/](src/python/vendor/openmatb/)
- External output root (used in runs): `C:\data\adaptive_matb\openmatb\...`
- Notes on questionnaires: NASA-TLX (genericscales) is a blocking plugin; scenario time pauses while completing it.

## Issues encountered (chronological)

### 1) Scenario path brittleness / scenario load failures
- Symptom: runs could fail depending on relative path interpretation.
- Fix implemented: wrapper resolves scenario paths robustly (repo-root-relative + validation) and writes an absolute path into OpenMATB config before launch.

### 2) Early aborts / missing markers when instructions block
- Symptom: if the session was exited early or the instruction screen blocked immediately, LSL/session markers could be missing or inconsistent.
- Fix implemented: ensure LSL starts and `STUDY/SESSION_START` marker is emitted before blocking instructions.

### 3) “Conditions started before instructions” (ordering within same timestamp)
- Symptom: tasks/conditions appeared to start before instructions even when scheduled “later”.
- Likely cause: multiple events at `0:00:00` can be executed in file-order; “start tasks at t=0” plus instructions at `0:00:01` can allow a brief initial update where tasks are live.
- Fix implemented: move instruction events to `0:00:00` and place them before task `start` events.

### 4) Tracking joystick-only input (mouse needed)
- Symptom: Tracking task was not controllable with mouse by default.
- Fix implemented (vendor): capture mouse motion deltas in the window and feed them as joystick-axis inputs in the scheduler.
- Follow-up improvement: increase mouse sensitivity and apply mouse deltas even when a joystick is present (so labs can test without physically disconnecting devices).

### 5) Perceived “blank start” / low event density
- Symptom: early parts of blocks felt empty; workload changes were subtle.
- Fix implemented: start tasks at `t=0` (visible immediately after instructions) and increase deterministic event density (especially in HIGH).

### 6) Resources (ResMan) state did not reset between blocks
- Symptom: “water” levels / pump states carried over between conditions.
- Fix implemented: explicit tank/pump resets at the start of each block.

### 7) Vendor scheduling crash (automaticsolver edge case)
- Symptom: crash in vendor scheduling when an automaticsolver event list was all-false.
- Fix implemented: vendor patch and submodule pointer bump.

### 8) LSL warnings in console
- Symptom: multicast bind warnings (IPv6 loopback) from LSL (`udp_server.cpp`).
- Impact: noisy logs; did not block runs.
- Action: treat as informational unless it correlates with missing streams.

## Operational notes
- When running with blocking screens (instructions/questionnaires), scenario time pauses; markers scheduled at the same scenario timestamp may log immediately upon resume.
- For input testing, ensure the OpenMATB window has focus; mouse events will not fire reliably when unfocused.

  - whether to require click-to-capture vs always-on mouse motion
