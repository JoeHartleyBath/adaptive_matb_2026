## Work done

### Bug fix: signal quality always 0 (`mwl_raw` stuck at 0.5)

Root cause identified: ANT eego amplifiers stream EEG in **Volts** (not µV). Raw signal RMS is ~0.008 V for clean EEG. The quality gate threshold was `0.5 < rms < 200.0`, which is µV-scale — so every window failed.

Fix:
- Moved quality check to **raw decimated data** (before filtering), threshold `1e-4 < rms < 100.0` (V-scale).
- Added **filter prewarm**: `prewarm=window_raw[:, 0]` passed to `initialize_filters()`, which scales IIR initial conditions by each channel's first sample to eliminate onset transients.
- Removed dead `preprocessor` variable that was initialised but never used.
- Added 5-window `[DIAG]` INFO log (temporary diagnostic).

Files changed: `src/eeg/eeg_filters.py`, `src/eeg/eeg_preprocessor.py`, `src/mwl_estimator.py`.

### New CLI flags in `run_full_study_session.py`

- `--skip-smoke-test`: skips the 15-second dual-EEG inlet smoke test before Phase 6.
- `--no-pause`: makes all `_pause()` interactive ENTER prompts auto-continue — useful for fast re-runs.
- Unicode `→` replaced with `->` in 4 places (was crashing on Windows cp1252 console).
- `_check_venv()` moved to after `args = parser.parse_args()` so `_NO_PAUSE` is set before it runs.

### MWL sanity checks (S002)

Intentionally tested MWL responsiveness by **deliberately varying engagement/effort** — decoupled from task difficulty. Results were consistent with this:

| Check | Result | Interpretation |
|---|---|---|
| MWL easy epoch (0–60 s) mean | 0.421 | Higher — deliberately engaged early |
| MWL hard epoch (60–170 s) mean | 0.267 | Lower — deliberately relaxed mid-run |
| MWL vs tracking error (Pearson r) | 0.12 | Near zero — expected given effort was decoupled from task |
| MWL vs tank deviation (Pearson r) | -0.14 | Near zero — same reason |
| Assistance trigger | t=16 s, `mwl_smoothed=0.798` | Fired during genuine MWL spike; turned off at t=39 s when smoothed value had decayed to 0.32 — trigger logic working |
| Difficulty step (t=60 s) | MWL 0.471 → 0.459 | No response — confirms effort was deliberately reduced at same time |

**Key caveat:** Model was calibrated on rest vs. task (binary: "doing MATB or not"), not on within-task load variation. Whether `mwl_raw` tracks cognitive load *within* a session is the core validation question; needs proper correlation data from a real effort-locked run.

**Plan:** Run the full 8-minute session properly (normal engagement, no effort manipulation) into a new session slot. Compare MWL, performance, and difficulty across the full run.
