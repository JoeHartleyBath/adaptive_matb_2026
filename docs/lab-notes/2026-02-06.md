## What changed today (high level)

### Scenario generation and invariants

- ResMan pump failures now follow the vendor-consistent invariant: `failure` then `off` exactly +10s.
- Added a strict event “edge buffer”: no scheduled demand events in the first/last 5 seconds.
- SysMon scale failures can now include scales 3–4 (F3/F4), matching the intended task.
- Moderate/High ResMan leak rates were reduced (small offsets) to stay “very difficult but not impossible”.
- Pump failure selection is “impact-balanced” across pumps to reduce block-to-block variance in effective difficulty.

### Operator workflow

- Added a block-start modal dialog that is participant-friendly and helps operator ID-check.
  - Full IDs only on the first block; compact ID footer for subsequent blocks.
- Added a single runner option for the attended within-session ladder check:
  - `--calibration-trend` runs `LOW → MODERATE → HIGH`.

### Outputs and interpretation

- `summarise_openmatb_performance.py` now emits derived communications SDT counts and a warning for the common “all MISS / NaN response fields” failure mode (usually caused by not pressing **ENTER** to validate a comms response).
- Operator checklist updated to explicitly remind about the comms “validate/submit” behavior.

### Submodule cleanliness

- The wrapper stages scenarios/instructions into the vendor OpenMATB include paths at runtime (required for vendor validation), and now **cleans them up afterwards** so `src/python/vendor/openmatb` does not remain dirty.
