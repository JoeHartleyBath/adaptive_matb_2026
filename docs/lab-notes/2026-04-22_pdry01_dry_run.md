## Bugs found and fixed

### Bug 1 — Baseline scenario permanent deadlock

**File:** `src/vendor/openmatb/plugins/abstractplugin.py`  
**Function:** `BlockingPlugin.update()`

**Cause:** `rest_baseline.txt` uses `instructions;allowkeypress;False` to show a
fixation cross without any on-screen text prompting SPACE.  Because `allowkeypress`
is `False`, the `SPACE` key check is never reached, so `go_to_next_slide` is never
set, and the scenario clock freezes permanently.

The `allowkeypress;False` flag was originally added to suppress the SPACE prompt on
familiarisation instruction slides.  It should not apply to the fixation-cross slide
where advance is purely time-based.

**Fix applied:** In `BlockingPlugin.update()`, after displaying each slide, if
`allowkeypress == False`, immediately set `self.go_to_next_slide = True`.  This
causes the slide to advance on the next clock tick rather than waiting for SPACE,
which is the correct behaviour for a timed fixation cross.

```python
# abstractplugin.py — inside BlockingPlugin.update(), after slide display:
if self.parameters['allowkeypress'] == False:
    self.go_to_next_slide = True
```

**Verification:** The rest_baseline scenario will now advance automatically.
Familiarisation slides (which do prompt SPACE) still work correctly because their
`allowkeypress` defaults to `True`.

## Offline recovery — `_tmp_calibrate_from_jsonl.py`

A temporary script was written to train the participant model from the JSONL
calibration files.  It reuses all the production preprocessing and training logic
(`build_mwl_training_dataset.py` helpers, `calibrate_participant.py` functions)
with only a JSONL-parsing front-end.

### Technical complications during development

Three issues were found and resolved while writing and running the script:

**Issue A — Channel count mismatch (132 vs 128)**  
JSONL `stream_info` records store only basic metadata; the full LSL channel
descriptor XML is not saved.  Without a descriptor, `_merge_eeg_streams` falls back
to taking all 66 channels per eego amp (64 EEG + 1 TRG + 1 CNT), yielding 132
instead of the expected 128 "ref" channels.  
Fix: inject a synthetic channel descriptor into the stream dict for 66-channel EEG
streams so the ref-channel filter works correctly.

**Issue B — Wrong marker stream selected**  
Three "Markers"-type LSL streams are present in a JSONL recording: two eego hardware
TRG streams and one OpenMATB stream.  `_find_stream(streams, "Markers")` returns the
first match, which is always a TRG stream (no MATB block events).  
Fix: prefer the stream whose name contains "openmatb" (case-insensitive).

**Issue C — Clock domain mismatch (EEG vs OpenMATB timestamps ~867,000 s apart)**  
The eego amplifiers and OpenMATB use different clock origins.  In XDF recordings,
`pyxdf` normalises all streams onto a common clock using the recorded clock-sync
samples.  The JSONL writer does not record clock-sync data.  
Fix: the JSONL header stores `lsl_start_time = pylsl.local_clock()` at recording
start.  The first EEG sample in the JSONL was pulled from the inlet at approximately
that same moment.  Shifting EEG timestamps by `(lsl_start_time - eeg_ts[0])` maps
them into the same `local_clock()` domain as the OpenMATB markers.
The correction was ~+867,425 s; after correction timestamps aligned perfectly
(block spans of ~7,375 samples at 128 Hz, matching the 57.6 s block duration).

## Issues / risks flagged for live study

1. **LabRecorder must be open before `--start-phase` resume.**  
   The cheat sheet already documents this but the current code silently falls back to
   JSONL with no user-visible warning that XDF recording is absent.  Consider adding
   a console warning when `--labrecorder-rcs` is set but LabRecorder is not found.

2. **Rest baseline baseline fixed but not yet tested end-to-end.**  
   The `allowkeypress` fix was applied; the fixation cross should now auto-advance.
   Verify in the next dry run (PDRY02) before any real participant.

3. **JSONL fallback is now recoverable** (via `_tmp_calibrate_from_jsonl.py`) but
   remains an operational risk — it adds a post-session recovery step that shouldn't
   be needed.  Primary mitigation: ensure LabRecorder is open before every session
   start and every resume.

4. **Clock correction in `_tmp_calibrate_from_jsonl.py` relies on
   `lsl_start_time ≈ first EEG sample pulled`.**  The first pulled chunk may lag
   by up to one buffer cycle (~0.1 s at 120 s max_buflen).  This introduces a small
   systematic offset in block boundary alignment but is negligible relative to the
   57.6 s block duration and 2 s analysis window.
