# Design choice: Full multitask MATB(-II) task set

**Decision**  
Use the full multitasking MATB(-II)-style configuration (i.e., running all subtasks) rather than a reduced one- or two-subtask configuration.

**Rationale**  
- The study objective is a multitasking MATB configuration, which requires running all subtasks.
- Reduced configurations (one or two subtasks) can induce high workload but introduce configuration-specific constraints.
- A recent review notes that multitasking MATB configurations are the configurations that have been validated in comparison to simulated flight environments (Caldwell, 1999; Caldwell et al., 2004; Bouak et al., 2019; as summarised in Pontiggia et al., 2024).


**Alternatives considered**  
- One-subtask MATB configuration.
- Two-subtask MATB configuration.

**Implications**  
Scenario design and implementation must ensure all subtasks are present/enabled for experimental blocks; workload manipulations should be expressed via difficulty/event structure within this multitask configuration.

**Status**  
Final

**References**  
- Pontiggia et al., 2024 (review of MATB(-II)-style paradigms for mental workload assessment; summarises validation and constraints across configurations)
